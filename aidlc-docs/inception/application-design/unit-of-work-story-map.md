## 新規ユーザーストーリー（Unit追加分）

### Unit-01: サウンドシステム

#### US-23: BGMの再生
**カジュアルゲーマーとして**、ゲーム中にBGMが流れてほしい。**なぜなら**、音楽があるとゲームの雰囲気に没入できるから。

**受入基準**:
- [ ] タイトル画面で軽快なBGMが再生される
- [ ] ゲームプレイ中はテンポの速い戦闘BGMに切り替わる
- [ ] ゲームオーバー時にゲームオーバージングルが再生される
- [ ] BGMはチップチューン（8bit）風のサウンド
- [ ] BGM音量は設定画面で調整可能（US-26と連携）

#### US-24: 効果音の再生
**カジュアルゲーマーとして**、射撃や敵撃破時に効果音が鳴ってほしい。**なぜなら**、音のフィードバックがあると操作の手応えが増して爽快感が高まるから。

**受入基準**:
- [ ] 射撃時に短い電子音が鳴る
- [ ] 敵撃破時に爆発音が鳴る
- [ ] アイテム取得時に上昇音が鳴る
- [ ] バフ発動時にキラキラ音が鳴る
- [ ] 仲間化成功時にファンファーレ音が鳴る
- [ ] ボス出現時に警告音が鳴る
- [ ] 防衛ライン突破時にダメージ音が鳴る
- [ ] SE音量は設定画面で調整可能（US-26と連携）

#### US-25: モバイルでの音声再生
**カジュアルゲーマーとして**、スマホでもBGMと効果音が正しく再生されてほしい。**なぜなら**、モバイルでも音ありでプレイしたいから。

**受入基準**:
- [ ] 初回タップ時にAudioContextが有効化される（ブラウザ自動再生ポリシー対応）
- [ ] AudioContext有効化前は無音でゲームが正常動作する
- [ ] iOS Safari、Android Chromeで音声が再生される

### Unit-02: 設定画面 & 遊び方ヘルプ

#### US-26: 設定画面
**カジュアルゲーマーとして**、タイトル画面からゲーム設定を変更したい。**なぜなら**、音量や操作方法を自分好みに調整したいから。

**受入基準**:
- [ ] タイトル画面に「SETTINGS」ボタンが表示される
- [ ] 設定画面がオーバーレイで表示される
- [ ] BGM音量をスライダーで調整できる（0〜100、デフォルト70）
- [ ] SE音量をスライダーで調整できる（0〜100、デフォルト80）
- [ ] 操作タイプを3択から選べる（ボタン / スワイプ / 両方）
- [ ] 設定値がlocalStorageに保存され、次回起動時に復元される
- [ ] 「閉じる」ボタンで設定画面を閉じてタイトル画面に戻れる

#### US-27: 遊び方ヘルプ
**カジュアルゲーマーとして**、ゲームの遊び方をいつでも確認したい。**なぜなら**、操作方法やアイテムの効果がわからない時にすぐ調べたいから。

**受入基準**:
- [ ] 設定画面内に「遊び方」ボタンが表示される
- [ ] 遊び方ヘルプ画面で以下の情報が閲覧できる:
  - 操作方法（PC: キーボード、モバイル: ボタン/スワイプ）
  - ゲームルール（防衛ライン、HP、ゲームオーバー条件）
  - 敵タイプ（通常/高速/タンク/ボスの特徴）
  - アイテム・バフ（4種バフの色・効果）
  - 武器（前方射撃/拡散/貫通の特徴）
  - 仲間化（仕組みと仲間の挙動）
- [ ] スクロールまたはページ送りで全内容を閲覧可能
- [ ] 「戻る」ボタンで設定画面に戻れる

## INVEST基準検証（新規ストーリー）

| 基準 | 検証結果 |
|------|---------|
| **Independent** | Unit-01はCore以外に依存なし。Unit-02はUnit-01のAPI（音量制御）に依存するが、スタブで先行開発可能 |
| **Negotiable** | 音量デフォルト値・SE種類数等はconfig変更で調整可能 |
| **Valuable** | US-23〜25: ゲーム体験の大幅向上。US-26〜27: ユーザビリティ向上 |
| **Estimable** | Web Audio API生成は事前調査済み、各ストーリーの工数見積もり可能 |
| **Small** | US-23〜25は各1〜2日、US-26〜27は各1〜2日で実装可能 |
| **Testable** | 全ストーリーに具体的な受入基準あり |
