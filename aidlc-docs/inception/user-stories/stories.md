# ユーザーストーリー

**プロジェクト**: ラストウォー型ミニゲーム風シューティングゲーム
**ペルソナ**: カジュアルゲーマー（ユウキ）
**作成日**: 2026-04-07（Iteration 2更新: 2026-04-09）
**総数**: 22件
**変更概要**: Iteration 2でXP/レベルアップ制をアイテムドロップ制に全面変更、ヒットカウント制導入、仲間化システム追加

---

## エピック1: ゲーム開始体験

### US-01: タイトル画面の表示
**カジュアルゲーマーとして**、ゲームを開くとタイトル画面が表示されてほしい。**なぜなら**、ゲームの雰囲気を把握し、準備ができてからプレイを開始したいから。

**受入基準**:
- [ ] ゲームタイトルが画面中央に表示される
- [ ] 「ゲーム開始」ボタンが明確に表示される
- [ ] 簡単な操作説明が表示される
- [ ] タイトル画面はPC/モバイル両方で正しくレイアウトされる

**関連要件**: FR-06

---

### US-02: ゲームの開始
**カジュアルゲーマーとして**、「ゲーム開始」ボタンを押すとすぐにゲームが始まってほしい。**なぜなら**、長いロードや手順なしにすぐ遊びたいから。

**受入基準**:
- [ ] 「ゲーム開始」ボタンをタップ/クリックするとPLAYING状態に遷移する
- [ ] ゲーム開始時にプレイヤーが画面下部中央に配置される
- [ ] HP 100、初期武器（前方射撃）が装備された状態で開始する
- [ ] 経過時間が0:00からカウント開始する

**関連要件**: FR-04, FR-06

---

## エピック2: コアゲームプレイ

### US-03: プレイヤーの左右移動
**カジュアルゲーマーとして**、画面下部で左右に移動したい。**なぜなら**、左右から来る敵の進行を防ぎ、全ての敵を倒し切りたいから。

**受入基準**:
- [ ] PC: A/D または左右矢印キーで左右移動できる
- [ ] モバイル: 左右ボタンタップまたは水平スワイプで移動できる
- [ ] 移動速度は基準値200px/秒（論理解像度基準）
- [ ] 画面端を超えて移動できない

**関連要件**: FR-01, NFR-02

---

### US-04: オート射撃
**カジュアルゲーマーとして**、攻撃は自動で行われてほしい。**なぜなら**、移動に集中しながら手軽に敵を倒したいから。

**受入基準**:
- [ ] プレイヤーは自動的に真上方向へ射撃を行う
- [ ] 初期武器（前方射撃）の発射間隔は0.15秒
- [ ] 弾丸が敵に命中するとヒットカウントが1減少する
- [ ] 射撃エフェクト（弾丸の発射アニメーション）が表示される
- [ ] 弾丸は銃口位置から発射される

**関連要件**: FR-01, FR-02-A

---

### US-05: 敵の出現と接近
**カジュアルゲーマーとして**、敵が画面上方から大群で押し寄せてきてほしい。**なぜなら**、大量の敵をなぎ倒す爽快感を味わいたいから。

**受入基準**:
- [ ] 敵は画面上方（X: 100〜620px範囲）からスポーンする
- [ ] 敵は下方向に移動する
- [ ] 通常敵・高速敵・タンク敵が定義されたパラメータで出現する
- [ ] ウェーブ3以降は2〜3体が同時にスポーンする（最大5体/回）
- [ ] 画面内の敵が300体に達した場合、新規スポーンが一時停止する
- [ ] 敵撃破時にエフェクトが表示される

**関連要件**: FR-02-B, FR-02-C, NFR-01

---

### US-06: 防衛ライン突破ダメージ
**カジュアルゲーマーとして**、敵を画面下部に到達させる前に倒さないとダメージを受けたい。**なぜなら**、迫り来る全ての敵を倒すゲームであり、撃ち漏らしにペナルティがあることで緊張感が生まれるから。

**受入基準**:
- [ ] 敵が防衛ライン（画面下端から32px上）を通過するとプレイヤーがダメージを受ける
- [ ] ダメージ値は敵タイプにより異なる（通常10、高速8、タンク15、ボス30）
- [ ] ラインを通過した敵は消滅する
- [ ] HPバーがダメージに応じて減少する
- [ ] 敵がプレイヤーに直接接触してもダメージは受けない

**関連要件**: FR-01, FR-02-B

---

### US-07: ヒットカウント表示
**カジュアルゲーマーとして**、敵の頭上に残りヒット数が数字で表示されてほしい。**なぜなら**、あと何発で倒せるかが一目でわかり、撃ち込む楽しさが増すから。

**受入基準**:
- [ ] 敵スプライトの上部に残りヒット数が白文字+黒縁で表示される
- [ ] 弾丸命中時にカウントが1減少する（攻撃UPバフ時は2減少）
- [ ] カウントが0になると敵が撃破される
- [ ] 被弾時に数字が赤く点滅する
- [ ] フォントサイズは通常敵20px、ボス28px
- [ ] 描画はビットマップフォント（事前レンダリング済み数字スプライト）を使用する

**関連要件**: FR-02-A

---

### US-08: HUD表示
**カジュアルゲーマーとして**、ゲーム中に自分のステータスを確認したい。**なぜなら**、残りHPやバフ状態を把握して戦略を立てたいから。

**受入基準**:
- [ ] HPバーが画面上部に表示される
- [ ] アクティブなバフのアイコン+残り時間バーがHPバーの下に表示される
- [ ] 現在のウェーブ/経過時間が表示される
- [ ] 仲間数（現在/最大）が表示される
- [ ] 現在装備中の武器アイコンが表示される
- [ ] HUD要素はゲームプレイを邪魔しない位置に配置される

**関連要件**: FR-05

---

## エピック3: アイテムドロップ・成長

### US-09: アイテムドロップと回収
**カジュアルゲーマーとして**、敵を倒すとアイテムが飛び出して自動的に集まってきてほしい。**なぜなら**、テンポよくアイテムを拾ってどんどん強くなりたいから。

**受入基準**:
- [ ] 敵撃破時に確率でパワーアップアイテムがドロップする（通常30%、高速35%、タンク50%、ボス100%で2〜3個）
- [ ] アイテムはマグネット半径1500px内でプレイヤーに自動的に引き寄せられる（速度500px/秒）
- [ ] プレイヤーの回収半径80px内に入るとアイテムが回収される
- [ ] 未回収アイテムは10秒で消滅する（残り3秒で点滅警告）
- [ ] 各アイテムタイプは色・形で視覚的に区別できる

**関連要件**: FR-03-A

---

### US-10: パワーアップバフの取得
**カジュアルゲーマーとして**、アイテムを拾うと一時的にパワーアップしてほしい。**なぜなら**、短時間の強化状態で敵をなぎ倒す瞬間が最高に気持ちいいから。

**受入基準**:
- [ ] パワーアップアイテム取得時に5秒間のバフが発動する
- [ ] 攻撃力UP（赤）: 1弾で2カウント減算
- [ ] 発射速度UP（黄）: 発射間隔が0.5倍（2倍速射撃）
- [ ] 移動速度UP（青）: 移動速度が1.5倍
- [ ] 弾幕モード（紫）: 弾数3倍+拡散角度拡大
- [ ] 同種バフ再取得で効果時間を5秒にリセットする（加算ではない）
- [ ] 異種バフは同時に適用可能
- [ ] バフ発動時にプレイヤー周囲にエフェクトが表示される

**関連要件**: FR-03-B

---

### US-11: 武器アイテムの取得
**カジュアルゲーマーとして**、敵がドロップした武器を拾って装備を切り替えたい。**なぜなら**、武器が変わると攻撃の手応えが変わり新鮮に遊べるから。

**受入基準**:
- [ ] 敵撃破時に5%の確率で武器アイテムがドロップする（パワーアップとは別枠判定）
- [ ] 武器アイテム取得で即座に装備武器が切り替わる（恒久的）
- [ ] 前方射撃: 連射速度 高（間隔0.15秒）、正面直線
- [ ] 拡散射撃: 連射速度 中（間隔0.25秒）、扇状3〜5発同時
- [ ] 貫通弾: 連射速度 低（間隔0.4秒）、敵を貫通
- [ ] 武器アイテムは武器の形をしたアイコンで表示される

**関連要件**: FR-03-C, FR-02-D

---

### US-12: 敵の仲間化
**カジュアルゲーマーとして**、倒した敵が仲間になって一緒に戦ってくれてほしい。**なぜなら**、仲間がどんどん増えて軍団になっていく感覚が爽快だから。

**受入基準**:
- [ ] 敵撃破時に確率で仲間に転換する（通常10%、高速8%、タンク5%、ボス0%）
- [ ] 最大仲間数は10体（上限到達時は仲間化判定を行わない）
- [ ] 仲間はプレイヤーの左右に動的間隔で配置される（1〜4体: 110px固定、5体以上: 画面幅に応じて縮小、最小40px）
- [ ] 仲間はプレイヤーに追従移動する
- [ ] 仲間は前方射撃でオート攻撃する
- [ ] 仲間はダメージを受けない
- [ ] 仲間化時に敵の色が変化→プレイヤー側に移動する演出がある

**関連要件**: FR-03-D

---

### US-13: 仲間の時間経過強化
**カジュアルゲーマーとして**、仲間が時間と共に成長して強くなってほしい。**なぜなら**、長く生き残るほど軍団が強力になり、爽快感が増していくから。

**受入基準**:
- [ ] 仲間は参加後10秒ごとに連射速度+10%（発射間隔10%短縮）
- [ ] 最大+100%（発射間隔半減）まで強化される
- [ ] 各仲間の強化は個別に計測される（参加タイミングが異なる）
- [ ] 強化は視覚的にわかるフィードバックがある（任意）

**関連要件**: FR-03-D

---

## エピック4: ゲーム進行

### US-14: ウェーブの進行
**カジュアルゲーマーとして**、時間経過とともに難易度が上がってほしい。**なぜなら**、段階的にチャレンジが増えることで飽きずに楽しめるから。

**受入基準**:
- [ ] ウェーブ1（0:00〜0:45）: 通常敵のみ、間隔1.0秒、1体/回
- [ ] ウェーブ2（0:45〜1:30）: 通常＋高速、間隔0.7秒、1〜2体/回
- [ ] ウェーブ3（1:30〜3:00）: 全タイプ混合、間隔0.5秒、2〜3体/回
- [ ] 以降30秒ごとに間隔0.05秒減少（下限0.15秒）、敵ヒット数+10%/30秒（ベース値比率、端数切り上げ）
- [ ] 現在のウェーブ/経過時間がHUDに表示される

**関連要件**: FR-02-C, FR-04

---

### US-15: ボス戦
**カジュアルゲーマーとして**、定期的に強力なボスと戦いたい。**なぜなら**、ボスを倒した時の達成感が大きく、ゲームのクライマックスを感じられるから。

**受入基準**:
- [ ] 1分30秒経過後に最初のボスが出現する
- [ ] 以降1分30秒ごとにボスが出現する
- [ ] ボスは他の敵より大型で視覚的に区別できる
- [ ] ボスのヒット数: 100（ウェーブ進行で+10%/30秒増加）
- [ ] ボス撃破時は100%でパワーアップアイテムを2〜3個ドロップする
- [ ] ボスは仲間化しない（仲間化率0%）

**関連要件**: FR-02-B, FR-02-C

---

## エピック5: ゲーム終了・リプレイ

### US-16: ゲームオーバー
**カジュアルゲーマーとして**、HPが0になるとゲームオーバー画面が表示されてほしい。**なぜなら**、自分の成果を確認してからリトライしたいから。

**受入基準**:
- [ ] HPが0になるとGAME_OVER状態に遷移する
- [ ] 生存時間が表示される
- [ ] 倒した敵の総数が表示される
- [ ] 獲得した仲間数が表示される

**関連要件**: FR-04, FR-05

---

### US-17: リトライ
**カジュアルゲーマーとして**、ゲームオーバー後にすぐリトライしたい。**なぜなら**、「あと1回だけ」の気持ちでもう一度挑戦したいから。

**受入基準**:
- [ ] ゲームオーバー画面に「リトライ」ボタンが表示される
- [ ] リトライボタンを押すとTITLE状態に遷移する
- [ ] タイトルから再度ゲーム開始で初期状態からプレイできる
- [ ] リトライまでの操作が2タップ以内

**関連要件**: FR-04, FR-05

---

## エピック6: クロスプラットフォーム

### US-18: モバイルでの快適プレイ
**カジュアルゲーマーとして**、スマートフォンでも快適にプレイしたい。**なぜなら**、通勤中や外出先でも手軽に遊びたいから。

**受入基準**:
- [ ] 左右移動ボタンが画面下部左右に配置される
- [ ] 水平スワイプでも移動操作が可能
- [ ] タッチ領域が最低44x44pt以上確保される
- [ ] モバイルで30FPS以上を維持する（敵150体以上で）
- [ ] 縦型レイアウト（9:16アスペクト比）で表示される

**関連要件**: NFR-01, NFR-02, NFR-05

---

### US-19: PCでの快適プレイ
**カジュアルゲーマーとして**、PCブラウザでも快適にプレイしたい。**なぜなら**、大きな画面で迫力のあるゲーム体験を楽しみたいから。

**受入基準**:
- [ ] キーボード（A/D、左右矢印）で移動操作できる
- [ ] 60FPSを維持する（敵300体で）
- [ ] レターボックス方式で正しいアスペクト比が維持される
- [ ] ブラウザリサイズ時にCanvasが再計算される
- [ ] モバイル専用のタッチUIが表示されない

**関連要件**: NFR-01, NFR-02, NFR-05

---

### US-20: パフォーマンスの維持
**カジュアルゲーマーとして**、大量の敵が出現しても滑らかに動いてほしい。**なぜなら**、カクカクすると操作が困難になりストレスを感じるから。

**受入基準**:
- [ ] 同時敵数300体でもPC 60FPSを維持する
- [ ] 同時描画オブジェクト800個でも処理落ちしない
- [ ] 弾丸同時表示は200発を上限とする（仲間の弾含む）
- [ ] アイテム同時表示は50個を上限とする
- [ ] 各上限到達時は適切に制御される（スポーン抑制等）

**関連要件**: NFR-01

---

### US-21: エラー時の復旧
**カジュアルゲーマーとして**、エラーが発生してもゲームが完全に壊れないでほしい。**なぜなら**、エラーでプレイが中断されるとフラストレーションを感じるから。

**受入基準**:
- [ ] JavaScriptエラー発生時にゲームが安全に停止する
- [ ] エラー画面にリロードボタンが表示される
- [ ] アセット読み込み失敗時はフォールバック表示で続行できる
- [ ] ブラウザタブ非アクティブ後の復帰時にゲームが正常に再開する

**関連要件**: NFR-08, FR-04

---

### US-22: バランスパラメータの調整容易性
**カジュアルゲーマーとして**、ゲームバランスが継続的に改善されてほしい。**なぜなら**、遊び続ける中でバランスの良いゲームは長く楽しめるから。

**受入基準**:
- [ ] 敵ステータス、ドロップ確率、バフ効果値、ウェーブ定義等のバランスパラメータがJSON設定ファイルで管理される
- [ ] コード変更なしでバランス調整が可能
- [ ] 設定ファイル変更後にビルド・デプロイで反映される

**関連要件**: NFR-03

## INVEST基準検証

| 基準 | 検証結果 |
|------|---------|
| **Independent** | 各ストーリーは独立して実装・テスト可能（US-10〜13はUS-09のアイテムドロップ基盤に依存するが、ゲームフローとして自然な依存） |
| **Negotiable** | 受入基準の数値パラメータはJSON設定で調整可能（US-22） |
| **Valuable** | 各ストーリーがプレイヤーに直接的な価値を提供 |
| **Estimable** | 受入基準が具体的で、開発工数の見積もりが可能 |
| **Small** | 各ストーリーは1〜3日程度で実装可能な粒度 |
| **Testable** | 全ストーリーに定量的な受入基準があり、テストケースに変換可能 |

## ストーリーマップ

```
ゲーム開始 → コアプレイ        → アイテム・成長      → ゲーム進行 → ゲーム終了
US-01,02    US-03〜08          US-09〜13             US-14,15    US-16,17
            (移動,射撃,敵,      (ドロップ,バフ,武器,     (ウェーブ,
             防衛,ヒット数,HUD)   仲間化,仲間強化)       ボス)

                  クロスプラットフォーム: US-18〜22（横断的）
```
