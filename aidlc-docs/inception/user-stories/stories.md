# ユーザーストーリー

**プロジェクト**: ラストウォー型ミニゲーム風シューティングゲーム
**ペルソナ**: カジュアルゲーマー（ユウキ）
**作成日**: 2026-04-07
**総数**: 20件

---

## エピック1: ゲーム開始体験

### US-01: タイトル画面の表示
**カジュアルゲーマーとして**、ゲームを開くとタイトル画面が表示されてほしい。**なぜなら**、ゲームの雰囲気を把握し、準備ができてからプレイを開始したいから。

**受入基準**:
- [ ] ゲームタイトルが画面中央に表示される
- [ ] 「ゲーム開始」ボタンが明確に表示される
- [ ] 簡単な操作説明が表示される
- [ ] タイトル画面はPC/モバイル両方で正しくレイアウトされる

**関連要件**: FR-06

---

### US-02: ゲームの開始
**カジュアルゲーマーとして**、「ゲーム開始」ボタンを押すとすぐにゲームが始まってほしい。**なぜなら**、長いロードや手順なしにすぐ遊びたいから。

**受入基準**:
- [ ] 「ゲーム開始」ボタンをタップ/クリックするとPLAYING状態に遷移する
- [ ] ゲーム開始時にプレイヤーが画面下部中央に配置される
- [ ] HP 100、初期武器（前方射撃）が装備された状態で開始する
- [ ] 経過時間が0:00からカウント開始する

**関連要件**: FR-04, FR-06

---

## エピック2: コアゲームプレイ

### US-03: プレイヤーの左右移動
**カジュアルゲーマーとして**、画面下部で左右に移動したい。**なぜなら**、左右から来る敵の進行を防ぎ、全ての敵を倒し切りたいから。

**受入基準**:
- [ ] PC: A/D または左右矢印キーで左右移動できる
- [ ] モバイル: 左右ボタンタップまたは水平スワイプで移動できる
- [ ] 移動速度は基準値200px/秒（論理解像度基準）
- [ ] 画面端を超えて移動できない
- [ ] 移動中もキャラクターのアニメーションが表示される

**関連要件**: FR-01, NFR-02

---

### US-04: オート射撃
**カジュアルゲーマーとして**、攻撃は自動で行われてほしい。**なぜなら**、移動に集中しながら手軽に敵を倒したいから。

**受入基準**:
- [ ] プレイヤーは自動的に射撃を行う
- [ ] 射程内の最も近い敵を優先ターゲットにする
- [ ] 射程内に敵がいない場合は正面方向に発射する
- [ ] 弾丸が敵に命中するとダメージを与える
- [ ] 射撃エフェクト（弾丸の発射アニメーション）が表示される

**関連要件**: FR-01

---

### US-05: 敵の出現と接近
**カジュアルゲーマーとして**、敵が画面上方から大群で押し寄せてきてほしい。**なぜなら**、大量の敵を撃退する爽快感を味わいたいから。

**受入基準**:
- [ ] 敵は画面上方および左右上部からスポーンする
- [ ] 敵はプレイヤーに向かって移動する
- [ ] 通常敵・高速敵・タンク敵が定義されたパラメータで出現する
- [ ] 時間経過でスポーン頻度が増加する
- [ ] 敵撃破時にエフェクトが表示される

**関連要件**: FR-02

---

### US-06: 防衛ライン突破ダメージ
**カジュアルゲーマーとして**、敵を画面下部に到達させる前に倒さないとダメージを受けたい。**なぜなら**、迫り来る全ての敵を倒すゲームであり、撃ち漏らしにペナルティがあることで緊張感が生まれるから。

**受入基準**:
- [ ] 敵が防衛ライン（画面下端から32px上）を通過するとプレイヤーがダメージを受ける
- [ ] ダメージ値は敵タイプにより異なる（通常10、高速8、タンク15、ボス30）
- [ ] ラインを通過した敵は消滅する
- [ ] HPバーがダメージに応じて減少する
- [ ] 敵がプレイヤーに直接接触してもダメージは受けない

**関連要件**: FR-01, FR-02

---

### US-07: HUD表示
**カジュアルゲーマーとして**、ゲーム中に自分のステータスを確認したい。**なぜなら**、残りHPやレベルアップの進捗を把握して戦略を立てたいから。

**受入基準**:
- [ ] HPバーが画面上部に表示される
- [ ] XPバー（次レベルまでの進捗）が表示される
- [ ] 現在のウェーブ/経過時間が表示される
- [ ] 所持武器と仲間の数が表示される
- [ ] HUD要素はゲームプレイを邪魔しない位置に配置される

**関連要件**: FR-05

---

## エピック3: 成長・強化

### US-08: 経験値の獲得と回収
**カジュアルゲーマーとして**、敵を倒すとXPアイテムが出現し、自動で回収されてほしい。**なぜなら**、テンポよく成長を感じたいから。

**受入基準**:
- [ ] 敵を倒すとXPアイテム（光る宝石）がドロップする
- [ ] XPドロップ量は敵タイプにより異なる（通常10、高速15、タンク25、ボス200）
- [ ] プレイヤーの回収半径48px以内で自動回収される
- [ ] 回収時にXPバーが増加する
- [ ] XPバーが満タンになるとレベルアップが発動する

**関連要件**: FR-03

---

### US-09: レベルアップ選択
**カジュアルゲーマーとして**、レベルアップ時に3つの強化オプションから選択したい。**なぜなら**、自分のプレイスタイルに合った成長をしたいから。

**受入基準**:
- [ ] レベルアップ時にゲームが一時停止する（敵・弾丸停止）
- [ ] 3つの強化オプションが表示される
- [ ] 選択肢は未取得武器・未最大レベルスキルからランダム抽選される
- [ ] 同一選択肢内に重複がない
- [ ] タップ/クリックで選択するとゲームが再開する
- [ ] 全強化取得済みの場合はHP 30%回復が代替表示される
- [ ] レベルアップ時にHP 10%回復する

**関連要件**: FR-03, FR-04

---

### US-10: 武器の獲得と強化
**カジュアルゲーマーとして**、新しい武器を手に入れたり、既存武器を強化したい。**なぜなら**、攻撃方法のバリエーションが増えると戦略の幅が広がるから。

**受入基準**:
- [ ] レベルアップ選択肢に新武器（拡散射撃、貫通弾）が出現する
- [ ] 武器は最大3種類まで同時装備可能（初期武器含む）
- [ ] 各武器は最大レベル5まで強化可能
- [ ] 強化によりダメージ増加・発射速度上昇・弾数増加する
- [ ] 装備中の武器が視覚的にわかる

**関連要件**: FR-03

---

### US-11: 仲間の追加
**カジュアルゲーマーとして**、仲間キャラクターを追加して一緒に戦いたい。**なぜなら**、火力が増えて多くの敵を倒せるようになると嬉しいから。

**受入基準**:
- [ ] レベルアップ選択肢に仲間追加オプションが出現する
- [ ] 仲間はプレイヤーの左右に等間隔（32px）で配置される
- [ ] 最大4体（左右各2体）まで追加可能
- [ ] 仲間はプレイヤーに追従移動する
- [ ] 仲間は前方射撃でオート攻撃する
- [ ] 仲間はダメージを受けない（プレイヤーのみ被ダメージ）

**関連要件**: FR-03

---

### US-12: パッシブスキルの獲得
**カジュアルゲーマーとして**、パッシブスキルでキャラクターを強化したい。**なぜなら**、移動速度やHP、攻撃力を上げて生存率を高めたいから。

**受入基準**:
- [ ] レベルアップ選択肢にパッシブスキルが出現する
- [ ] 移動速度UP（+10%/レベル）が適用される
- [ ] 最大HP UP（+20/レベル）が適用される
- [ ] 攻撃力UP（+15%/レベル）が全武器に適用される
- [ ] XP獲得量UP（+20%/レベル）が回収時に適用される
- [ ] 各スキルは最大レベル5

**関連要件**: FR-03

---

## エピック4: ゲーム進行

### US-13: ウェーブの進行
**カジュアルゲーマーとして**、時間経過とともに難易度が上がってほしい。**なぜなら**、段階的にチャレンジが増えることで飽きずに楽しめるから。

**受入基準**:
- [ ] ウェーブ1（0:00〜1:00）: 通常敵のみ、スポーン間隔2.0秒
- [ ] ウェーブ2（1:00〜2:30）: 通常敵＋高速敵、スポーン間隔1.5秒
- [ ] ウェーブ3（2:30〜4:30）: 全通常タイプ混合、スポーン間隔1.0秒
- [ ] 以降30秒ごとにスポーン間隔0.05秒減少、敵HP+5%
- [ ] 現在のウェーブ/経過時間がHUDに表示される

**関連要件**: FR-02, FR-04

---

### US-14: ボス戦
**カジュアルゲーマーとして**、定期的に強力なボスと戦いたい。**なぜなら**、ボスを倒した時の達成感が大きく、ゲームのクライマックスを感じられるから。

**受入基準**:
- [ ] 2分経過後に最初のボスが出現する
- [ ] 以降2分ごとにボスが出現する
- [ ] ボスは他の敵より大型で視覚的に区別できる
- [ ] ボスのHP: 500（ウェーブ進行で増加）
- [ ] ボス撃破時にXP 200をドロップする
- [ ] ウェーブ進行に伴いボスのHP・ダメージが増加する

**関連要件**: FR-02

---

## エピック5: ゲーム終了・リプレイ

### US-15: ゲームオーバー
**カジュアルゲーマーとして**、HPが0になるとゲームオーバー画面が表示されてほしい。**なぜなら**、自分の成果を確認してからリトライしたいから。

**受入基準**:
- [ ] HPが0になるとGAME_OVER状態に遷移する
- [ ] 生存時間が表示される
- [ ] 倒した敵の総数が表示される
- [ ] 到達レベルが表示される
- [ ] ゲームオーバー画面が表示されるまでに短い演出がある

**関連要件**: FR-04, FR-05

---

### US-16: リトライ
**カジュアルゲーマーとして**、ゲームオーバー後にすぐリトライしたい。**なぜなら**、「あと1回だけ」の気持ちでもう一度挑戦したいから。

**受入基準**:
- [ ] ゲームオーバー画面に「リトライ」ボタンが表示される
- [ ] リトライボタンを押すとTITLE状態に遷移する
- [ ] タイトルから再度ゲーム開始で初期状態からプレイできる
- [ ] リトライまでの操作が2タップ以内

**関連要件**: FR-04, FR-05

---

## エピック6: クロスプラットフォーム

### US-17: モバイルでの快適プレイ
**カジュアルゲーマーとして**、スマートフォンでも快適にプレイしたい。**なぜなら**、通勤中や外出先でも手軽に遊びたいから。

**受入基準**:
- [ ] 左右移動ボタンが画面下部左右に配置される
- [ ] 水平スワイプでも移動操作が可能
- [ ] タッチ領域が最低44x44pt以上確保される
- [ ] モバイルで30FPS以上を維持する（敵100体以上で）
- [ ] 縦型レイアウト（9:16アスペクト比）で表示される

**関連要件**: NFR-01, NFR-02, NFR-05

---

### US-18: PCでの快適プレイ
**カジュアルゲーマーとして**、PCブラウザでも快適にプレイしたい。**なぜなら**、大きな画面で迫力のあるゲーム体験を楽しみたいから。

**受入基準**:
- [ ] キーボード（A/D、左右矢印）で移動操作できる
- [ ] 60FPSを維持する（敵200体で）
- [ ] レターボックス方式で正しいアスペクト比が維持される
- [ ] ブラウザリサイズ時にCanvasが再計算される
- [ ] モバイル専用のタッチUIが表示されない

**関連要件**: NFR-01, NFR-02, NFR-05

---

### US-19: パフォーマンスの維持
**カジュアルゲーマーとして**、大量の敵が出現しても滑らかに動いてほしい。**なぜなら**、カクカクすると操作が困難になりストレスを感じるから。

**受入基準**:
- [ ] 同時敵数200体でもPC 60FPSを維持する
- [ ] 同時描画オブジェクト500個でも処理落ちしない
- [ ] 同時敵数が200体を超えた場合はスポーン抑制またはデスポーンで制御する
- [ ] 弾丸同時表示は100発を上限とする

**関連要件**: NFR-01

---

### US-20: エラー時の復旧
**カジュアルゲーマーとして**、エラーが発生してもゲームが完全に壊れないでほしい。**なぜなら**、エラーでプレイが中断されるとフラストレーションを感じるから。

**受入基準**:
- [ ] JavaScriptエラー発生時にゲームが安全に停止する
- [ ] エラー画面にリロードボタンが表示される
- [ ] アセット読み込み失敗時はフォールバック表示で続行できる
- [ ] ブラウザタブ非アクティブ後の復帰時にゲームが正常に再開する

**関連要件**: NFR-08, FR-04

## INVEST基準検証

| 基準 | 検証結果 |
|------|---------|
| **Independent** | 各ストーリーは独立して実装・テスト可能（US-09はUS-08に依存するが、これはゲームフローとして自然な依存） |
| **Negotiable** | 受入基準の数値パラメータは調整可能（バランス調整の余地あり） |
| **Valuable** | 各ストーリーがプレイヤーに直接的な価値を提供 |
| **Estimable** | 受入基準が具体的で、開発工数の見積もりが可能 |
| **Small** | 各ストーリーは1〜3日程度で実装可能な粒度 |
| **Testable** | 全ストーリーに定量的な受入基準があり、テストケースに変換可能 |
