# Requirements Document

## Intent Analysis

- **User Request**: ラストウォー（Last War）のミニゲーム風の、敵の大群を撃ちまくるシューティング型ゲーム
- **Request Type**: New Project（新規プロジェクト）
- **Request Clarity**: Standard（会話で明確化済み）
- **Scope Estimate**: Multiple Components（複数コンポーネント）
- **Complexity Estimate**: Moderate（中程度の複雑さ）
- **Requirements Depth**: Standard
- **Reference Game**: Last War: Survival Game のミニゲーム（シューティング型）

---

## 1. プロジェクト概要

### 1.1 ゲームコンセプト
ラストウォー（Last War）のミニゲーム風ディフェンスシューティングゲーム。プレイヤーは画面下部で左右に移動しながら、上方から大量に押し寄せてくる敵をオート射撃で迎撃する。敵が画面下部の防衛ラインを突破するとダメージを受けるため、全ての敵を到達前に撃破することが目標。敵を倒して経験値を獲得し、レベルアップで武器強化・仲間追加・パッシブスキルを選択して強化していく。

### 1.2 ターゲットプラットフォーム
- **プラットフォーム**: Webブラウザ（HTML5/Canvas）
- **対応デバイス**: PC（キーボード/マウス）、モバイル（タッチ操作）
- **技術スタック**: HTML5, Canvas API, JavaScript/TypeScript

---

## 2. 機能要件

### FR-01: プレイヤー操作
- **視点**: トップダウン（見下ろし）
- **位置**: 画面下部に配置
- **移動**: 左右移動のみ（モバイル：左右ボタンまたは水平スワイプ、PC：A/D または 左右矢印キー）
- **移動速度**: 基準値 200px/秒（論理解像度基準）、パッシブスキルで最大1.5倍まで増加
- **攻撃**: オート射撃（最も近い敵を優先ターゲット。射程内に敵がいない場合は正面方向へ発射）
- **グラフィック**: 2Dカラフルドット絵スタイル
- **HP**: 初期値 100。最大HP はパッシブスキルで増加可能
- **被ダメージ条件**: 敵がプレイヤーの防衛ライン（画面下端）に到達・通過した場合にダメージ（敵タイプ別ダメージ値はFR-02で定義）。敵との直接接触ではダメージを受けない
- **防衛ライン**: 画面下端から32px上の位置に設定。敵がこのラインを通過するとダメージが発生し、その敵は消滅する
- **回復手段**: 初期実装ではレベルアップ時にHP 10%回復。将来的に回復アイテムを検討
- **プレイヤー衝突判定**: 弾丸の発射元判定用の円形コリジョン（プレイヤー半径: 16px、論理解像度基準）。敵との接触ダメージはなし

### FR-02: 敵システム
- 初期は基本的な4種類の敵
  - **通常敵**: HP 20、移動速度 80px/秒、突破ダメージ 10、XPドロップ 10
  - **高速敵**: HP 10、移動速度 160px/秒、突破ダメージ 8、XPドロップ 15
  - **タンク敵**: HP 60、移動速度 40px/秒、突破ダメージ 15、XPドロップ 25
  - **ボス敵**: HP 500（ウェーブにより増加）、移動速度 30px/秒、突破ダメージ 30、XPドロップ 200
- **スポーン位置**: 画面上方および左右上部（画面外からスポーン）
- **スポーン間隔**: 初期 2.0秒、ウェーブ進行に伴い最小 0.3秒まで減少
- **ウェーブ定義**:
  - ウェーブ1（0:00〜1:00）: 通常敵のみ、スポーン間隔 2.0秒
  - ウェーブ2（1:00〜2:30）: 通常敵＋高速敵、スポーン間隔 1.5秒
  - ウェーブ3（2:30〜4:30）: 全通常タイプ混合、スポーン間隔 1.0秒
  - 以降30秒ごとにスポーン間隔 0.05秒減少、敵HP +5%
- **ボス出現条件**: 2分経過後、以降2分ごとに出現。ウェーブ進行に伴いHP・ダメージ増加
- **衝突判定**: 円形コリジョン（通常敵半径: 12px、タンク敵: 20px、ボス敵: 40px）
- 後から新しい敵タイプを追加しやすい設計

### FR-03: 成長・強化システム
- **経験値（XP）システム**: 敵を倒すとXPドロップ（光る宝石アイテム）、プレイヤーに一定距離以内で自動回収（回収半径: 48px）
- **レベルアップ選択**: レベルアップ時に3つの強化オプションから選択（ゲーム一時停止）
  - **選択肢生成ルール**: 未取得武器・未最大レベルスキルからランダム抽選。同一選択肢内の重複なし
  - **同一スキル再出現**: 最大レベル未到達なら再出現可能
  - **全強化取得済みの場合**: HP回復（30%回復）を代替選択肢として表示
- **武器カテゴリ**（初期実装）:
  - 前方射撃（初期装備）: 正面に弾丸発射
  - 拡散射撃: 扇状に複数弾発射
  - 貫通弾: 敵を貫通する弾丸
- **武器同時装備上限**: 3種（初期武器含む）
- **武器強化**: 各武器最大レベル5（ダメージ増加、発射速度上昇、弾数増加など）
- **仲間追加**: プレイヤーの左右に仲間キャラクターが追加され、一緒にオート射撃
  - **最大追加数**: 左右各2体、計4体まで
  - **配置方式**: プレイヤーの左右に等間隔配置（間隔: 32px）、プレイヤーに追従移動
  - **仲間の武器**: プレイヤーの初期武器と同じ前方射撃（プレイヤーの武器強化は適用されない）
  - **仲間のHP**: なし（被ダメージしない、プレイヤーのみがダメージ判定対象）
- **パッシブスキルカテゴリ**（各最大レベル5）:
  - 移動速度UP（+10%/レベル）
  - 最大HP UP（+20/レベル）
  - 攻撃力UP（+15%/レベル）
  - XP獲得量UP（+20%/レベル）

### FR-04: ゲームフロー
- **エンドレスサバイバルモード**: 死ぬまで生き残る
- 時間経過で難易度が段階的に上昇（敵の数・速度・体力が増加）
- 定期的にボスが出現
- プレイヤーの HP が 0 になるとゲームオーバー
- ゲームオーバー時にスコア表示（生存時間、倒した敵数、到達レベル）
- **ゲーム状態一覧と遷移**:
  - `TITLE` → ゲーム開始ボタン押下 → `PLAYING`
  - `PLAYING` → レベルアップ条件達成 → `LEVEL_UP`（ゲーム一時停止、敵・弾丸停止）
  - `LEVEL_UP` → 強化選択完了 → `PLAYING`
  - `PLAYING` → HP が 0 → `GAME_OVER`
  - `GAME_OVER` → リトライ押下 → `TITLE`
- **タブ非アクティブ時**: ゲームループを一時停止（requestAnimationFrame自動停止に依存）。復帰時はデルタタイム上限を設けてスキップ防止
- **ブラウザリサイズ時**: Canvasサイズを再計算し、レターボックスを再適用

### FR-05: UI/HUD
- HP バー（画面上部）
- XP バー（次レベルまでの進捗）
- 現在のウェーブ/経過時間表示
- 所持武器・仲間の表示
- レベルアップ選択画面（ゲーム一時停止、3択程度の強化選択）
- ゲームオーバー画面（スコア表示、リトライボタン）

### FR-06: タイトル画面
- ゲームタイトル表示
- 「ゲーム開始」ボタン
- 簡単な操作説明

## 3. 非機能要件

### NFR-01: パフォーマンス
- **PC目標**: 60 FPS を維持（同時表示敵数200体でも）
- **モバイル目標**: 30 FPS 以上を維持（同時表示敵数100体以上で）
- **同時描画オブジェクト上限目標**: 500個（敵・弾丸・エフェクト・XPアイテム含む）
- **同時敵数上限**: 200体（これを超える場合は古い敵をデスポーンまたはスポーン抑制）
- **弾丸同時表示上限**: 100発
- **Canvas 2D描画で性能不足の場合**: OffscreenCanvas / WebGL への段階的移行を検討

### NFR-02: レスポンシブ対応
- PC とモバイル両方で遊べるレスポンシブ設計
- **モバイル操作UI**:
  - 左右移動ボタン: 画面下部左右に配置（タップ長押しで連続移動）
  - 水平スワイプ: 画面下半分の任意の位置でスワイプ操作も可能
  - 攻撃: オート（操作不要）
- **HUD配置ガイドライン**:
  - 上部: HPバー、XPバー、ウェーブ/時間表示
  - 下部: 左右移動ボタン（モバイルのみ表示）
  - タッチ領域は最低 44x44pt 以上を確保（Apple HIG準拠）

### NFR-03: 拡張性
- 新しい敵タイプを容易に追加できる設計
- 新しい武器・スキルを容易に追加できる設計
- 将来的なステージ追加が可能な設計

### NFR-04: グラフィックス
- 2Dカラフルドット絵スタイル
- アニメーション: 移動、射撃エフェクト、敵撃破エフェクト

### NFR-05: 画面仕様
- **基準解像度**: 720x1280（縦型）
- **アスペクト比**: 9:16（縦型、モバイルファースト）
- **スケーリング方式**: レターボックス（基準アスペクト比を維持し、余白は黒帯で埋める）
- **Canvas描画**: 論理解像度720x1280で描画し、表示時にデバイス解像度にスケーリング

### NFR-06: デプロイ・ホスティング
- **ホスティング**: 静的サイトホスティング（GitHub Pages または Cloudflare Pages 推奨）
- **HTTPS**: 必須（ホスティングサービスの標準機能を利用）
- **ビルド・デプロイ**: npm scripts によるビルド、Git push トリガーの自動デプロイ
- **CDN**: ホスティングサービス標準のCDNを利用

### NFR-07: セキュリティ
- **入力バリデーション**: 移動入力値の範囲チェック（フレームあたり最大移動量を制限）、不正な入力値は無視
- **ゲーム状態整合性**: レベルアップ選択データはゲーム内部状態と照合し、不正な強化選択を防止
- **コード難読化**: 本番ビルド時にminify実施（難読化は任意）
- **依存ライブラリ管理**: npm audit を CI/CD に組み込み、高リスク脆弱性はリリース前に対応
- **CSP（Content Security Policy）**: インラインスクリプト制限、外部リソース読み込み制限を設定
- **対応ブラウザ**: Chrome / Safari / Firefox の最新2バージョン（モバイル含む）

### NFR-08: エラーハンドリング
- **グローバルエラーハンドラー**: window.onerror / unhandledrejection でキャッチし、ゲーム状態を安全に停止
- **ゲームループエラー**: try-catch でラップし、致命的エラー時はエラー画面を表示（リロードボタン付き）
- **アセット読み込み失敗**: フォールバック表示（プレースホルダー描画）で続行可能にする

### NFR-09: CI/CD・品質管理
- **CI/CD**: GitHub Actions による自動ビルド・デプロイ（mainブランチへのpush時）
- **リンター**: ESLint による静的解析をCIに組み込み
- **テスト**: ユニットテスト（Jest等）をCIに組み込み、主要ゲームロジックをカバー
- **パフォーマンス計測**: 開発時にFPSカウンター表示機能を組み込み（デバッグモード）
