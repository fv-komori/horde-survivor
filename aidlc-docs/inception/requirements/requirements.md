# Requirements Document

## Intent Analysis

### プロジェクトの全体像
ラストウォー（Last War: Survival Game）のミニゲームをモデルにした、Webブラウザ向けディフェンスシューティングゲームの開発。プレイヤーは画面下部で左右に移動しながら、上方から押し寄せる大量の敵をオート射撃で迎撃する。

### 開発の経緯
1. **Iteration 1**（2026-04-07）: 基本ゲームの新規開発。ECS（Entity-Component-System）アーキテクチャ + TypeScript + Canvas 2Dで実装。XP経験値を貯めてレベルアップ時に武器強化・仲間追加・パッシブスキルを選択する成長システム。→ 実装完了・動作確認済み
2. **Iteration 2**（2026-04-09・本要件）: 爽快感が不足していたため、ゲームプレイの大幅改修を実施。
   - **テンポ向上**: 敵の大量出現 + 射撃強化でなぎ倒し感を演出
   - **成長システム刷新**: XP/レベルアップ制を全面廃止し、アイテムドロップ制（一時バフ + 武器アイテム）に置き換え
   - **ヒットカウント制導入**: HP制を廃止し、敵の頭上に残りヒット数を数字表示。1弾=1カウント減算で連射が重要に
   - **仲間化システム**: 敵撃破時に確率で仲間に転換（最大10体）、時間経過で連射速度が向上

### 現在の要件属性
- **Request Type**: Enhancement（Iteration 1の大幅改善）
- **Request Clarity**: Standard（会話で明確化済み）
- **Scope Estimate**: Multiple Components（成長システム・敵・UI・バランス全体）
- **Complexity Estimate**: Moderate（既存ECSアーキテクチャ内での変更）
- **Requirements Depth**: Standard

## 1. プロジェクト概要

### 1.1 ゲームコンセプト
ラストウォー（Last War）のミニゲーム風ディフェンスシューティングゲーム。プレイヤーは画面下部で左右に移動しながら、上方から大量に押し寄せてくる敵をオート射撃で迎撃する。敵が画面下部の防衛ラインを突破するとダメージを受けるため、全ての敵を到達前に撃破することが目標。**敵を倒すとパワーアップアイテムがドロップし、拾うことで一時的に強化される。倒した敵が確率で仲間になり、時間経過と共に軍団が成長して爽快感が増していく。**

### 1.2 ターゲットプラットフォーム
- **プラットフォーム**: Webブラウザ（HTML5/Canvas）
- **対応デバイス**: PC（キーボード/マウス）、モバイル（タッチ操作）
- **技術スタック**: HTML5, Canvas API, TypeScript

## 2. 機能要件

### FR-01: プレイヤー操作 ※変更なし
- **視点**: トップダウン（見下ろし）
- **位置**: 画面下部に配置
- **移動**: 左右移動のみ（モバイル：左右ボタンまたは水平スワイプ、PC：A/D または 左右矢印キー）
- **移動速度**: 基準値 200px/秒（論理解像度基準）
- **攻撃**: オート射撃（常に真上方向へ発射）
- **グラフィック**: 2Dカラフルドット絵スタイル（兵士風デザイン）
- **HP**: 初期値 100
- **被ダメージ条件**: 敵が防衛ライン（画面下端から32px上）を通過した場合にダメージ
- **プレイヤー衝突判定**: 円形コリジョン（半径72px）

### FR-02: 敵システム 【変更: テンポ向上 + ヒットカウント制】

#### FR-02-A: ヒットカウント制（ダメージシステム変更）
- **概要**: 従来のHP制を廃止し、ヒットカウント制に変更。敵の頭上に残りヒット数を数字で表示し、弾が当たるたびに1ずつ減少。0になると撃破
- **1弾=1カウント**: 弾丸1発で必ずカウントが1減る（武器による差はなし）
- **攻撃UPバフ時**: 1弾で2カウント減算（FR-03-B参照）
- **頭上数字表示**: 敵スプライトの上部に白文字+黒縁で残りヒット数を表示。被弾時に数字が赤く点滅
- **数字サイズ**: フォントサイズ20px（ボスは28px）、常時表示
- **描画最適化方針**: ヒット数テキストはCanvas fillTextではなく、事前レンダリング済み数字スプライト（ビットマップフォント）を使用し描画負荷を軽減する。画面外の敵のヒット数描画は省略する

#### FR-02-B: 敵タイプ定義
- 初期は基本的な4種類:
  - **通常敵**: ヒット数 5, 移動速度 100px/秒, 突破ダメージ 10, 仲間化率 10%
  - **高速敵**: ヒット数 2, 移動速度 200px/秒, 突破ダメージ 8, 仲間化率 8%
  - **タンク敵**: ヒット数 15, 移動速度 50px/秒, 突破ダメージ 15, 仲間化率 5%
  - **ボス敵**: ヒット数 100（ウェーブにより増加）, 移動速度 30px/秒, 突破ダメージ 30, 仲間化率 0%

#### FR-02-C: スポーン・ウェーブ
- **スポーン位置**: 画面上方（X: 100〜620px範囲）
- **スポーン間隔**: 初期 1.0秒（※旧2.0秒から短縮）、ウェーブ進行で最小 0.15秒まで減少
- **同時出現数**: 初期1体/回、ウェーブ3以降は2〜3体/回同時スポーン
- **ウェーブ定義**:
  - ウェーブ1（0:00〜0:45）: 通常敵のみ、間隔1.0秒、1体/回
  - ウェーブ2（0:45〜1:30）: 通常＋高速、間隔0.7秒、1〜2体/回
  - ウェーブ3（1:30〜3:00）: 全タイプ混合、間隔0.5秒、2〜3体/回
  - 以降30秒ごとに間隔0.05秒減少、敵ヒット数+10%/30秒（ベース値に対する比率増加、端数切り上げ）
- **同時スポーン数上限**: ウェーブ進行に伴う同時スポーン数は最大5体/回を上限とする。また、画面内の敵数がNFR-01の同時敵数上限（300体）に達した場合、新規スポーンを一時停止する
- **ボス出現条件**: 1分30秒経過後、以降1分30秒ごとに出現
- 後から新しい敵タイプを追加しやすい設計

#### FR-02-D: 武器の差別化（ヒットカウント制対応）
- **1弾=1カウント**のため、武器の差別化は**連射速度**が主軸:
  - **前方射撃**: 連射速度 高（間隔0.15秒）、弾道=正面直線
  - **拡散射撃**: 連射速度 中（間隔0.25秒）、弾道=扇状3〜5発同時、広範囲カバー
  - **貫通弾**: 連射速度 低（間隔0.4秒）、弾道=敵を貫通（複数体に同時ヒット可能）
- 連射が速い＝同じ時間でより多くカウントを削れる＝強い

### FR-03: アイテムドロップ・成長システム 【全面変更】

#### FR-03-A: アイテムドロップ基盤
- **ドロップ条件**: 敵撃破時にランダムでアイテムをドロップ
- **ドロップ確率**:
  - 通常敵: 30%
  - 高速敵: 35%
  - タンク敵: 50%
  - ボス敵: 100%（複数アイテム同時ドロップ: 2〜3個）
- **アイテム回収**: プレイヤーとの接触で自動回収（回収半径80px）
- **アイテムマグネット**: マグネット半径1500px、引き寄せ速度500px/秒
- **アイテム消滅**: ドロップ後10秒で消滅（点滅警告: 残り3秒）
- **アイテム描画**: 各アイテムタイプ別のアイコン（色・形で区別可能）

#### FR-03-B: パワーアップアイテム（一時バフ）
- **効果時間**: 5秒間
- **バフ重複**: 同種バフは効果時間を5秒にリセットする（残り時間への加算ではない）。異種バフは同時適用可能
- **バフ発動演出**: 取得時にプレイヤー周囲にエフェクト表示
- **バフ種類**:
  1. **攻撃力UP**: 1弾で2カウント減算（通常は1）、アイコン色=赤
  2. **発射速度UP**: 発射間隔0.5倍（2倍速射撃）、アイコン色=黄
  3. **移動速度UP**: 移動速度1.5倍、アイコン色=青
  4. **弾幕モード**: 弾数3倍+拡散角度拡大、アイコン色=紫
- **ドロップウェイト**: 攻撃力UP 30%, 発射速度UP 30%, 移動速度UP 20%, 弾幕モード 20%

#### FR-03-C: 武器アイテム
- **武器変更**: 武器アイテム取得で即座に装備武器が切り替わる（恒久的、一時バフではない）
- **武器タイプ**:
  1. **前方射撃（初期装備）**: 正面に弾丸発射
  2. **拡散射撃**: 扇状に複数弾発射
  3. **貫通弾**: 敵を貫通する弾丸
- **ドロップ確率**: パワーアップアイテムとは別枠。敵撃破時5%で武器アイテムドロップ
- **武器アイテム描画**: 武器の形をしたアイコン

#### FR-03-D: 仲間化システム 【新規】
- **仲間化条件**: 敵撃破時に一定確率で仲間に転換
  - 通常敵: 10%
  - 高速敵: 8%
  - タンク敵: 5%
  - ボス敵: 0%（仲間化しない）
- **最大仲間数**: 10体（上限到達時は仲間化判定を行わず、通常のアイテムドロップ判定のみ実施）
- **仲間配置**: プレイヤーの左右に等間隔配置。間隔はプレイヤー左右の配置可能幅（基準解像度720pxからプレイヤー幅とマージンを除いた範囲）を仲間数で等分して動的に決定する（最大110px、最小40px）。仲間数1〜4体: 110px固定、5体以上: `min(110, (配置可能幅 / 仲間数))` px。プレイヤーに追従
- **仲間の武器**: プレイヤーの初期武器と同じ前方射撃
- **仲間のHP**: なし（被ダメージしない）
- **仲間化演出**: 敵の色が変化→プレイヤー側に移動するアニメーション
- **時間経過強化**: 仲間は参加後10秒ごとに連射速度+10%（発射間隔を10%短縮、最大+100%=発射間隔半減）

### FR-04: ゲームフロー 【一部変更】
- **エンドレスサバイバルモード**: 死ぬまで生き残る
- 時間経過で難易度が段階的に上昇（敵の数・速度・体力が増加）
- 定期的にボスが出現
- プレイヤーの HP が 0 になるとゲームオーバー
- ゲームオーバー時にスコア表示（生存時間、倒した敵数、仲間数）
- **ゲーム状態一覧と遷移**:
  - `TITLE` → ゲーム開始ボタン押下 → `PLAYING`
  - `PLAYING` → HP が 0 → `GAME_OVER`
  - `GAME_OVER` → リトライ押下 → `TITLE`
  - ※ `LEVEL_UP`状態を廃止（レベルアップ選択画面なし）
- **タブ非アクティブ時**: ゲームループを一時停止
- **ブラウザリサイズ時**: Canvasサイズを再計算、レターボックス再適用

### FR-05: UI/HUD 【変更】

- **仲間数表示**: 現在の仲間数 / 最大数
- 所持武器表示（現在装備中の武器アイコン）
- ゲームオーバー画面（スコア表示、リトライボタン）
- ~~XPバー~~ → 廃止
- ~~レベルアップ選択画面~~ → 廃止

## 3. 非機能要件

### NFR-01: パフォーマンス 【調整】
- **PC目標**: 60 FPS を維持（同時表示敵数300体でも）← 大量敵対応で引き上げ
- **モバイル目標**: 30 FPS 以上を維持（同時表示敵数150体以上で）
- **同時描画オブジェクト上限目標**: 800個（敵・弾丸・エフェクト・アイテム含む）← 引き上げ
- **同時敵数上限**: 300体 ← 引き上げ
- **弾丸同時表示上限**: 200発 ← 引き上げ（仲間の弾含む）
- **アイテム同時表示上限**: 50個

### NFR-02: レスポンシブ対応 ※変更なし
- PC とモバイル両方で遊べるレスポンシブ設計
- モバイル操作UI（左右移動ボタン、水平スワイプ）
- タッチ領域最低44x44pt

### NFR-03: 拡張性 【強化】
- 新しい敵タイプを容易に追加できる設計
- 新しい武器・アイテムを容易に追加できる設計
- **バランスパラメータの外部設定化**: 敵ステータス、ドロップ確率、バフ効果値、ウェーブ定義等の数値パラメータはJSON設定ファイルで管理し、コード変更なしでバランス調整可能とする

### NFR-04: グラフィックス 【微変更】
- 2Dカラフルドット絵スタイル
- アニメーション: 移動、射撃エフェクト、敵撃破エフェクト
- **追加**: アイテムドロップアニメーション、バフ発動エフェクト、仲間化演出

### NFR-05: 画面仕様 ※変更なし
- 基準解像度: 720x1280（縦型）
- アスペクト比: 9:16
- レターボックス方式

### NFR-06: デプロイ・ホスティング ※変更なし
- 静的サイトホスティング
- HTTPS必須、npm scripts ビルド

### NFR-07: セキュリティ 【具体化】
- **CSP（Content Security Policy）**: `default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; connect-src 'none'` を基本ディレクティブとし、外部リソース読み込みを禁止
- **入力バリデーション**: ユーザー入力はキーボードイベント（移動キー）とタッチイベント（座標）のみ。座標値は画面範囲内にクランプし、不正な値は無視する
- **npm audit**: CI/CDパイプラインで `npm audit --audit-level=high` を実行し、high以上の脆弱性が検出された場合はビルドを失敗させる
- **クライアントサイド改ざん方針**: 本ゲームはクライアントサイド完結（サーバー通信なし）のため、DevTools等によるパラメータ改ざんは許容する。将来ランキング機能等を追加する場合は、サーバーサイドでのスコア検証が必要

### NFR-08: エラーハンドリング ※変更なし
- グローバルエラーハンドラー、ゲームループtry-catch

### NFR-09: CI/CD・品質管理 ※変更なし
- GitHub Actions、ESLint、Jest
