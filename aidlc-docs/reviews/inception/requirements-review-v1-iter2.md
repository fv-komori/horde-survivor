## 2. バックエンド開発者（B）

### レビュー観点
- ゲームロジックの整合性（数値バランス、計算式の妥当性）
- データ構造の設計（敵/武器/スキルの定義、拡張性）
- 処理フロー（ゲームループ、衝突判定、スポーンロジック）
- 拡張性（新要素追加時の影響範囲）

### OK項目
| # | 対象要件 | OK理由 |
|---|---------|--------|
| B-OK-1 | FR-02: 敵パラメータ定義 | 4種の敵にHP/速度/ダメージ/XPドロップが定量的に定義。衝突判定の半径も敵タイプ別に記載されており、ゲームロジック実装の入力として十分 |
| B-OK-2 | FR-02: ウェーブ定義 | ウェーブ1〜3の時間帯・敵構成・スポーン間隔が具体的に定義。ウェーブ4以降の漸進的難易度上昇（30秒ごとにスポーン間隔-0.05秒、敵HP+5%）もデータドリブンで実装可能 |
| B-OK-3 | FR-03: パッシブスキル定量定義 | 4種のパッシブスキルに具体的な数値（+10%/レベル等）と最大レベル（5）が定義。レベルアップ選択肢の生成ルール（未取得・未最大レベルからの抽選）も明確 |
| B-OK-4 | FR-01: 衝突判定 | 円形コリジョン方式で統一（プレイヤー16px、敵タイプ別12/20/40px）。距離計算ベースの衝突判定として実装がシンプルかつ高速 |
| B-OK-5 | NFR-01: 同時描画・敵数上限 | 同時敵数200体、弾丸100発、総描画500個の上限が定義。上限超過時の挙動（デスポーン/スポーン抑制）も記載されており、ゲームループの負荷管理設計が可能 |
| B-OK-6 | FR-03: 仲間システム | 仲間の最大数（4体）、配置方式（左右等間隔32px）、武器仕様（前方射撃固定、プレイヤー強化非適用）、HP（なし）が明確に定義。ロジック実装に曖昧さがない |
| B-OK-7 | NFR-03: 拡張性方針 | 新敵タイプ・武器・スキルの追加容易性、将来のステージ追加が要件として明記。FR-02の「後から新しい敵タイプを追加しやすい設計」も具体的方針として記載 |

### NG項目
| # | 対象要件 | NG理由 | 提案 |
|---|---------|--------|------|
| B-NG-1 | FR-03: **武器ダメージ・発射速度・射程の基準値未定義** | 武器カテゴリ（前方射撃/拡散射撃/貫通弾）は定義されているが、各武器の基本ダメージ・発射間隔・射程距離の数値が記載されていない。「ダメージ増加、発射速度上昇」とあるが基準値がなく、バランス調整の出発点がない | **提案**: FR-03に各武器の基準パラメータテーブルを追加（基本ダメージ、発射間隔(秒)、射程(px)、弾速(px/秒)、レベルアップ時の変化量） <!-- severity: important --> |
| B-NG-2 | FR-03: **XPレベルアップ必要量の計算式が未定義** | 経験値システムは定義されているが、レベルアップに必要なXP量（レベルごとの増加曲線）が記載されていない。ゲームバランスの核となる要素が不明 | **提案**: FR-03にレベルアップ必要XPの計算式を追加（例: 基準値 + レベル * 増分、または指数関数的増加の方針） <!-- severity: important --> |
| B-NG-3 | FR-02: **ウェーブ4以降のスポーン間隔下限到達後の挙動未定義** | スポーン間隔が最小0.3秒まで減少するとあるが、ウェーブ定義では30秒ごとに0.05秒減少。ウェーブ3終了時点で1.0秒、以降0.3秒に到達後（約7分経過）の難易度上昇手段が不明。敵HP+5%のみでは長時間プレイの難易度曲線が緩やかすぎる可能性 | **提案**: スポーン間隔下限到達後の代替難易度上昇手段を定義（例: 敵HP増加率の加速、新敵タイプ混入率の変化、同時スポーン数の増加など） <!-- severity: medium --> |
| B-NG-4 | FR-01: **プレイヤー移動のCanvas境界制限が未記載** | プレイヤーの左右移動について、画面端到達時の挙動（壁際で停止/ループ/バウンス）が記載されていない。衝突判定半径16pxと画面幅720pxの関係から移動範囲は16〜704pxとなるが、明示されていない | **提案**: FR-01に「プレイヤーは画面左端・右端で停止（コリジョン半径分のマージン）」等の境界条件を追記 <!-- severity: medium --> |
| B-NG-5 | FR-02: **敵の移動パターン（AI）が未定義** | 敵の移動速度は定義されているが、移動方向/パターン（直進のみか、プレイヤー追尾か、ランダム蛇行か）が記載されていない。ボス敵の行動パターンも移動速度のみで攻撃パターンがない | **提案**: FR-02に敵タイプ別の移動パターンを追加（通常敵: プレイヤー方向へ直進、高速敵: プレイヤー追尾、ボス: 特殊パターン等）。ボスの攻撃パターン（射撃/範囲攻撃等）の有無も明記 <!-- severity: important --> |
